FileReader 对象详解 ​

一、简介 ​

FileReader 对象允许 Web 应用程序异步读取存储在用户计算机上的文件（或原始数据缓冲区）的内容，使用 File 或 Blob 对象指定要读取的文件或数据。

可获取 File 对象的场景：

  • 用户在 <input type="file"> 元素上选择文件后，返回的 FileList 对象；
  • 拖放操作生成的 DataTransfer 对象；
  • 在 HTMLCanvasElement 上执行 mozGetAsFile() 方法后返回的结果。

重要限制：FileReader 仅用于安全地从用户（远程）系统读取文件内容，不能直接通过文件路径读取本地文件。若需按路径名读取文件，应使用服务器端 Ajax 方案（跨域需配置 CORS）。

二、属性 ​

属性描述
FileReader.readyState表示 FileReader 的状态（数字）：
- EMPTY：0（未加载任何数据）
- LOADING：1（数据加载中）
- DONE：2（读取请求完成）
FileReader.result读取的结果（格式由读取方法决定，如文本、DataURL、ArrayBuffer 等）
FileReader.error读取过程中产生的错误对象（error.code 含错误码：1-文件未找到、2-安全错误、3-读取中断、4-文件不可读、5-编码错误）

三、方法 ​

FileReader 实例有 4 个核心方法（用于读取文件或中断读取）。注意：读取结果不通过方法返回，而是存储在 result 属性中。

方法描述
FileReader.abort()中止读取操作，执行后 readyState 变为 DONE。
FileReader.readAsArrayBuffer(blob)读取 Blob 内容为 ArrayBuffer 格式，结果存于 result。
FileReader.readAsBinaryString(blob)读取 Blob 内容为原始二进制字符串，结果存于 result。
FileReader.readAsDataURL(blob)读取 Blob 内容为 DataURL（Base64 编码），结果存于 result（常用于图片预览）。
FileReader.readAsText(blob, [encoding])读取 Blob 内容为文本，可选指定编码（默认 UTF-8），结果存于 result。

四、事件处理 ​

FileReader 继承自 EventTarget，支持完整的事件模型，可通过 onxxx 回调或 addEventListener 监听。

事件触发时机
FileReader.onabort读取操作被中断时触发（如调用 abort()）。
FileReader.onerror读取操作发生错误时触发（错误信息在 reader.error 中）。
FileReader.onload读取操作成功完成时触发。
FileReader.onloadstart读取操作开始时触发。
FileReader.onloadend读取操作结束时触发（无论成功/失败）。
FileReader.onprogress读取 Blob 过程中持续触发（可用于显示进度）。

五、浏览器支持检测 ​

javascript
if (window.FileReader) {
  const fr = new FileReader();
  // 执行 FileReader 相关逻辑
} else {
  alert("您的浏览器不支持 FileReader！");
}

六、基础使用示例 ​

6.1 文本文件读取 + 图片预览 ​

html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <title>FileReader 示例</title>
</head>
<body>
  <!-- 文本文件选择 -->
  <input type="file" id="textFile" />
  <!-- 图片文件选择 + 预览 -->
  <input type="file" id="imgFile" />
  <img src="" id="imgPreview" style="max-width: 300px;" />

  <script>
    // 1. 文本文件读取
    const textFile = document.getElementById("textFile");
    textFile.onchange = function () {
      const reader = new FileReader();
      // 以文本形式读取文件
      reader.readAsText(this.files[0]);
      // 读取成功后回调
      reader.onload = function () {
        console.log("文本内容：", this.result);
      };
    };

    // 2. 图片预览（DataURL 形式）
    const imgFile = document.getElementById("imgFile");
    const imgPreview = document.getElementById("imgPreview");
    imgFile.onchange = function () {
      const reader = new FileReader();
      // 以 DataURL 形式读取图片
      reader.readAsDataURL(this.files[0]);
      // 读取成功后设置图片 src
      reader.onload = function () {
        imgPreview.src = reader.result;
        console.log("图片 Base64 编码：", reader.result);
      };
    };
  </script>
</body>
</html>

6.2 事件全流程监听 ​

html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <title>FileReader 事件示例</title>
</head>
<body>
  <input type="file" id="fileInput" />

  <script>
    const fileInput = document.getElementById("fileInput");
    fileInput.onchange = function () {
      const reader = new FileReader();
      reader.readAsText(this.files[0]);

      reader.onloadstart = function () {
        console.log("onloadstart：读取开始，readyState =", this.readyState); // 1
      };

      reader.onprogress = function (e) {
        if (e.lengthComputable) {
          const progress = (e.loaded / e.total) * 100;
          console.log(`onprogress：读取中，进度 ${progress.toFixed(2)}%`);
        }
      };

      reader.onload = function () {
        console.log("onload：读取成功，readyState =", this.readyState); // 2
        console.log("读取结果：", this.result);
      };

      reader.onerror = function () {
        console.log("onerror：读取错误，错误信息：", this.error);
      };

      reader.onabort = function () {
        console.log("onabort：读取被中断");
      };

      reader.onloadend = function () {
        console.log("onloadend：读取结束，readyState =", this.readyState); // 2
      };
    };
  </script>
</body>
</html>

七、Node.js 中操作文件（扩展） ​

浏览器中的 JavaScript 受安全限制，无法直接操作本地文件；但 Node.js 中的 JavaScript 支持文件操作（需使用 fs 模块）。

7.1 异步读取文件 ​

javascript
const fs = require("fs");

fs.readFile("example.txt", "utf8", function (err, data) {
  if (err) {
    console.error("读取失败：", err);
    return;
  }
  console.log("文件内容：", data);
});

7.3 顺序读取多个文件（Promise 封装） ​

javascript
const fs = require("fs");

function readFilePromise(filePath, encoding = "utf8") {
  return new Promise((resolve, reject) => {
    fs.readFile(filePath, encoding, (err, data) => {
      if (err) reject(err);
      else resolve(data);
    });
  });
}

// 顺序读取 a.txt → b.txt → c.txt
readFilePromise("./a.txt")
  .then((data) => {
    console.log("a.txt 内容：", data);
    return readFilePromise("./b.txt");
  })
  .then((data) => {
    console.log("b.txt 内容：", data);
    return readFilePromise("./c.txt");
  })
  .then((data) => {
    console.log("c.txt 内容：", data);
  })
  .catch((err) => {
    console.error("读取出错：", err);
  });

总结 ​

FileReader 是前端处理用户本地文件的核心工具，支持异步、多格式读取与细粒度事件监听，能优雅实现“文件预览”“文本解析”等场景；而 Node.js 的 fs 模块则填补了服务端文件操作的能力，二者结合可覆盖前后端文件处理需求。
