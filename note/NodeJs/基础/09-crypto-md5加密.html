MD5简介 ​

1.定义

MD5即Message-Digest Algorithm 5（中文名为消息摘要算法第五版）为计算机安全领域广泛使用

的一种散列函数，用以提供消息的完整性保护。

2.作用

MD5的作用是让大容量信息在用数字签名软件签署私人密钥前被”压缩”成一种保密的格式（就是把一

个任意长度的字节串变换成一定长的十六进制数字串）。除了MD5以外，其中比较有名的还有

sha-1、RIPEMD以及Haval等。

3.MD5算法特点

MD5算法具有以下特点：

(1) 压缩性：任意长度的数据，算出的MD5值长度都是固定的。

(2) 容易计算：从原数据计算出MD5值很容易。

(3) 抗修改性：对原数据进行任何改动，哪怕只修改1个字节，所得到的MD5值都有很大区别。

(4) 强抗碰撞：已知原数据和其MD5值，想找到一个具有相同MD5值的数据（即伪造数据）是非常困

难的。

Node.js使用MD5 ​

javascript
注: MD5是一种常用的哈希算法，用于给任意数据一个“签名”。这个签名通常用一个十六进制的字符串表示：

        const crypto = require('crypto'); //引入crypto加密模块

        const hash = crypto.createHash('md5'); //规定使用哈希算法中的MD5算法

        // 可任意多次调用update(),效果相当于多个字符串相加
        hash.update('123a');
        hash.update('456');

        //最终加密的字符串为'123a456',hash.digest('hex')表示输出的格式为16进制
        console.log(hash.digest('hex')); // 57d15a6ee827933b25dab42111b6803b

        update()方法默认字符串编码为UTF-8，也可以传入Buffer。

        注:

        如果要计算SHA1，只需要把'md5'改成'sha1'，就可以得到SHA1的结果

        3be8b1b4c1b081f65522c4dd69b1d5c5713b038b。当然还可以使用更安全的sha256和sha512。

目前单次的MD5加密并不安全,很容易被暴力破解,因此推荐多次加密和使用秘钥加密

javascript
    1.多次MD5加密

        例如要加密 '123' 这个字符串

           const crypto = require('crypto');

        function md5 (text) {
            return crypto.createHash('md5').update(text).digest('hex');
        };

        多次加密:

        var str='123';

        var pass=md5(str);

        var FinalPass=md5(pass);
多次MD5加密不容易被破解,但是嵌套起来也比较麻烦, 因此我再介绍另外一种方法
    2.MD5结合秘钥加密

        例如要加密 '123' 这个字符串

        const crypto = require('crypto');

        function md5 (text) {
            return crypto.createHash('md5').update(text).digest('hex');
        };

        结合秘钥加密:

        var str='123';

        var md5_suffix='asd1wdad1d哈do捱三顶五#&8qw'; //md5_suffix作为秘钥,越复杂越好

        var FinalPass=md5(str+md5_suffix);
结合秘钥使用MD5加密更难被破解,但是也有一定的缺陷,就是如果秘钥一旦丢失就无法进行验证

MD5加密语句的详细讲解 ​

javascript
    (1) crypto.createHash(algorithm)

        创建并返回一个hash对象，它是一个指定算法的加密hash，用于生成hash摘要。

        参数algorithm可选择系统上安装的OpenSSL版本所支持的算法。例如：'sha1', 'md5', 'sha256', 

        'sha512'等。在近期发行的版本中，openssl list-message-digest-algorithms会显示这些可用的摘要算法

    (2) hash.update(data)

        更新hash的内容为指定的data。当使用流数据时可能会多次调用该方法。

    (3) hash.digest(encoding='binary')

        计算所有传入数据的hash摘要。参数encoding（编码方式）可以为'hex', 'binary' 或者'base64'。

完整的案例 ​

js
// 引入Node.js的crypto模块  
const crypto = require('crypto');  
  
// 生成随机密钥和初始化向量的函数  
function generateKeyAndIV() {  
    const key = crypto.randomBytes(32); // 生成32字节的随机密钥（AES-256需要）  
    const iv = crypto.randomBytes(16); // 生成16字节的随机初始化向量  
    return { key, iv }; // 返回包含密钥和初始化向量的对象  
}  
  
// 加密函数  
function encrypt(text, key, iv) {  
    let cipher = crypto.createCipheriv('aes-256-cbc', Buffer.from(key), iv); // 创建加密器  
    let encrypted = cipher.update(text); // 加密输入文本（可能不包含最终块）  
    encrypted = Buffer.concat([encrypted, cipher.final()]); // 合并加密的文本和最终块  
    return { iv: iv.toString('hex'), encryptedData: encrypted.toString('hex') }; // 返回包含IV（十六进制）和加密数据（十六进制）的对象  
}  
  
// 解密函数  
function decrypt(text, key, iv) {  
    iv = Buffer.from(iv, 'hex'); // 将十六进制字符串的IV转换为Buffer  
    let decipher = crypto.createDecipheriv('aes-256-cbc', Buffer.from(key), iv); // 创建解密器  
    let decrypted = decipher.update(Buffer.from(text, 'hex')); // 解密输入文本（可能不包含最终块）  
    decrypted = Buffer.concat([decrypted, decipher.final()]); // 合并解密的文本和最终块  
    return decrypted.toString(); // 返回解密后的文本（字符串）  
}  
  
// 使用示例  
const { key, iv } = generateKeyAndIV(); // 生成密钥和IV  
const originalText = 'Hello, world!'; // 待加密的原始文本  
const encrypted = encrypt(originalText, key, iv); // 加密原始文本  
const decrypted = decrypt(encrypted.encryptedData, key, encrypted.iv); // 解密加密后的文本  
  
// 输出结果  
console.log('Original:', originalText); // 输出原始文本  
console.log('Encrypted:', encrypted); // 输出加密后的结果（包含IV和加密数据）  
console.log('Decrypted:', decrypted); // 输出解密后的文本

这段代码展示了如何在Node.js中使用crypto模块进行AES-256-CBC加密和解密。它首先生成一个随机的密钥和初始化向量，然后使用这些密钥和向量来加密和解密一个字符串。加密结果是一个包含加密数据和初始化向量的对象，解密过程则使用这个对象来恢复原始文本。

hex的意思 ​

在计算机科学中，HEX是十六进制的英文缩写。十六进制是一种基数为16的计数系统，采用0~9和A~F（或a~f，大小写均可）共16个符号来表示数值，其中A~F代表10~15。十六进制常用于计算机领域，因为它能够比十进制更简洁地表示二进制数，是二进制和十进制之间的一个很好的中间数制。例如，在计算机编程和硬件设计中，十六进制常用于表示内存地址、颜色代码、数据块等。

iv有什么用处 ​

在加密过程中，iv（初始化向量，Initialization Vector）扮演着非常重要的角色，尤其是在使用某些加密模式（如CBC、CFB、OFB等）时。这些模式要求除了密钥之外，还需要一个初始化向量来确保即使相同的明文块被多次加密，也会产生不同的密文块，从而增强加密的安全性。

具体来说，iv的用处包括：

  1. 随机性：iv是随机生成的，这确保了即使使用相同的密钥加密相同的明文，每次加密的结果也会因为iv的不同而不同。这有助于抵抗某些类型的攻击，比如重放攻击（replay attack）和某些类型的统计分析攻击。
  2. 模式要求：在如CBC（密码块链接）这样的加密模式中，iv与第一个明文块一起被加密，然后加密的结果与下一个明文块一起被用于后续的加密过程。这种机制确保了即使明文块之间有重复，加密后的密文块也不会相同。
  3. 可预测性：虽然iv本身应该是随机的，但在某些应用场景中（比如加密协议中的消息传递），接收方需要能够预测或至少验证iv的值。因此，iv通常是明文传输的，或者至少可以通过某种方式验证其真实性。然而，由于iv并不包含密钥信息，即使iv被泄露，也不会直接影响加密数据的安全性（只要密钥保持安全）。
  4. 性能：虽然iv对加密过程的安全性至关重要，但它本身并不增加加密算法的复杂度或降低其性能。实际上，使用iv可以使加密过程更加安全，同时保持原有的性能水平。

总之，iv在加密过程中用于提供额外的随机性，以确保即使加密相同的明文，每次加密的结果也会不同。这是通过使用不同的iv来实现的，从而增强了加密过程的安全性。在使用需要iv的加密模式时，必须为每个加密操作生成一个新的、随机的iv，并将其与加密数据一起存储或传输，以便在解密时能够使用。
