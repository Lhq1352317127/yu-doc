NestJS 核心概念：Middleware 中间件 ​

一、中间件基础 ​

中间件是 NestJS 中处理请求的重要环节，与 Express、Koa 中的中间件概念类似，可获取 Request、Response 对象和 next 函数，主要作用包括：

  • 执行任意代码
  • 对请求和响应进行拦截与修改
  • 结束请求-响应周期
  • 通过 next() 调用下一个中间件

注意：若当前中间件未结束请求-响应周期，必须调用 next() 函数，否则请求会处于挂起状态，阻塞整个应用。

二、中间件类型 ​

1. 类中间件 ​

（1）创建类中间件 ​

需使用 @Injectable() 装饰器，并实现 NestMiddleware 接口（必须包含 use 方法）：

typescript
// Logger.middleware.ts
import { Injectable, NestMiddleware } from "@nestjs/common";
import { Request, Response } from "express";

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: () => void) {
    console.log('logger middleware', `url: ${req.url}`);
    next(); // 调用下一个中间件
  }
}

（2）使用类中间件 ​

需在模块中挂载，模块需实现 NestModule 接口，通过 configure 方法配置：

typescript
// user.module.ts
import { Module, NestModule } from '@nestjs/common';
import { UserService } from './user.service';
import { UserController } from './user.controller';
import { LoggerMiddleware } from '../middleware/Logger.middleware';

@Module({
  controllers: [UserController],
  providers: [UserService]
})
export class UserModule implements NestModule {
  configure(consumer) {
    consumer
      .apply(LoggerMiddleware) // 挂载中间件
      .forRoutes(UserController); // 对指定控制器的所有路由生效
  }
}

2. 函数中间件 ​

函数中间件更简洁，无需类和装饰器，直接定义函数即可：

（1）创建函数中间件 ​

typescript
// logger.middleware.ts
import { Request, Response } from "express";

export function LoggerMiddleware(req: Request, res: Response, next: () => void) {
  console.log('logger middleware', `url: ${req.url}`);
  next();
}

三、全局中间件 ​

若需中间件对所有路由生效，可在入口文件 main.ts 中通过 app.use() 挂载：

typescript
// main.ts
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
import { LoggerMiddleware } from './middleware/logger.middleware';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.use(LoggerMiddleware); // 全局挂载中间件
  await app.listen(3000);
}
bootstrap();

四、中间件应用场景 ​

  • 日志系统：记录请求信息
  • CORS跨域处理：设置跨域响应头
  • 图片防盗链：验证请求来源（如 Referer 校验）
  • 身份验证前置处理：请求前验证token有效性
