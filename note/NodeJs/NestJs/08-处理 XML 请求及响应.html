Nest.js 处理 XML 请求及响应 ​

一、背景 ​

在使用 Nest.js 开发服务端程序时，可能会遇到需要处理 XML 格式请求与响应的场景。由于 Nest.js 底层默认使用 Express 及其 body-parser 中间件，而 body-parser 不原生支持 XML，因此需要额外配置以支持 application/xml 类型的请求。本文介绍如何通过中间件和自定义装饰器优雅地处理 XML 数据。

二、依赖选择 ​

  • body-parser：用于解析请求体，需配置支持 XML 类型。
  • fast-xml-parser：用于 XML 与 JSON 之间的转换（社区星数较高，对 TypeScript 支持较好）。

安装依赖：

bash
npm install body-parser fast-xml-parser
npm install @types/body-parser -D  # 类型声明（可选）

三、实现步骤 ​

1. 创建 XML 处理中间件 ​

通过中间件配置 body-parser 以解析 XML 格式的请求体。

（1）生成中间件 ​

bash
nest g middleware xml  # 生成 xml.middleware.ts
1

（2）实现中间件逻辑 ​

typescript
// src/common/xml.middleware.ts
import { Injectable, NestMiddleware } from '@nestjs/common';
import * as bodyParser from 'body-parser';
import { Request, Response, NextFunction } from 'express';

// 配置 body-parser 解析 XML 文本
const bodyParserMiddleware = bodyParser.text({
  limit: '1024kb',  // 限制请求体大小
  type: 'application/xml'  // 仅处理 XML 类型请求
});

@Injectable()
export class XMLMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    bodyParserMiddleware(req, res, next);  // 应用解析逻辑
  }
}

2. 在模块中配置中间件 ​

指定中间件作用的路由（如仅对 wework/* 路径的 POST 请求生效）。

typescript
// src/app.module.ts
import { Module, NestModule, MiddlewareConsumer, RequestMethod } from '@nestjs/common';
import { XMLMiddleware } from './common/xml.middleware';

@Module({
  // ...其他配置
})
export class AppModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer
      .apply(XMLMiddleware)  // 应用 XML 中间件
      .forRoutes({ 
        path: 'wework/*',  // 作用路径
        method: RequestMethod.POST  // 作用请求方法
      });
  }
}

3. 在控制器中解析 XML ​

通过 @Body() 获取 XML 字符串后，使用 fast-xml-parser 解析为 JSON。

typescript
// src/wework/wework.controller.ts
import { Controller, Post, Body, Logger } from '@nestjs/common';
import * as xmlParser from 'fast-xml-parser';

@Controller('wework')
export class WeworkController {
  private logger = new Logger(WeworkController.name);

  @Post('callback')
  respond(@Body() body: string) {
    // 验证 XML 格式合法性
    if (xmlParser.validate(body) !== true) {
      return 'xml invalid';
    }

    this.logger.log('XML 请求体:', body);
    // 解析 XML 为 JSON（解析结果默认包裹在 xml 属性中）
    const parsed = xmlParser.parse(body);
    
    // 处理业务逻辑（示例：调用服务层方法）
    return this.weworkService.respond(parsed.xml);
  }
}

4. 优化：自定义 XML 解析装饰器 ​

为避免在控制器中重复编写解析逻辑，可通过自定义装饰器封装 XML 解析过程。

（1）生成装饰器 ​

bash
nest g decorator xml  # 生成 xml.decorator.ts
1

（2）实现装饰器逻辑 ​

typescript
// src/common/xml.decorator.ts
import { createParamDecorator, ExecutionContext } from '@nestjs/common';
import * as xmlParser from 'fast-xml-parser';

export const XML = createParamDecorator((data: string, ctx: ExecutionContext) => {
  // 从请求中获取原始 XML 字符串
  const { body } = ctx.switchToHttp().getRequest();

  // 验证 XML 格式
  if (xmlParser.validate(body) !== true) {
    return 'xml invalid';
  }

  // 解析并返回 xml 节点内容
  const { xml } = xmlParser.parse(body);
  return xml;
});

（3）在控制器中使用装饰器 ​

typescript
// src/wework/wework.controller.ts
import { Controller, Post, Param, HttpCode } from '@nestjs/common';
import { XML } from '../common/xml.decorator';
import { IWeworkEncrypt } from './interfaces/wework.interface';  // 自定义类型

@Controller('wework')
export class WeworkController {
  @Post('callback/:uuid')
  @HttpCode(200)  // 自定义响应状态码
  async respond(
    @Param('uuid') uuid: string,
    @XML() parsed: IWeworkEncrypt  // 使用自定义装饰器直接获取解析后的数据
  ) {
    return this.weworkService.respond(uuid, parsed);
  }
}

四、总结 ​

处理 XML 请求的核心步骤：

  1. 通过中间件配置 body-parser 解析 XML 文本；
  2. 在模块中指定中间件作用的路由；
  3. 利用 fast-xml-parser 解析 XML 为 JSON；
  4. 通过自定义装饰器封装解析逻辑，简化控制器代码。

该方案适用于企业微信回调、第三方 XML 接口交互等场景，兼顾了 TypeScript 类型支持和代码复用性。
