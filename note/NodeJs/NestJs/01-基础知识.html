NestJS 基础知识与实战指南 ​

一、什么是 NestJS？ ​

NestJS 是一个用于构建高效、可扩展的 Node.js 服务器端应用程序的框架。它基于 TypeScript 构建，结合了 OOP（面向对象编程）、FP（函数式编程）和 FRP（函数响应式编程）的最佳实践，提供了类似 Angular 的模块化架构，让开发者能够轻松构建复杂应用。

核心特点： ​

  • 模块化架构：通过模块（Module）、控制器（Controller）和服务（Service）组织代码，职责清晰
  • 依赖注入：内置依赖注入系统，简化组件间通信和测试
  • TypeScript 优先：原生支持 TypeScript，提供类型安全和更好的开发体验
  • 兼容多种 HTTP 框架：默认使用 Express，可无缝切换到 Fastify 以提升性能
  • 丰富的生态系统：内置支持 RESTful API、GraphQL、WebSocket 等，并有大量第三方模块

四、核心概念 ​

4.1 模块（Module） ​

模块是 Nest 应用的基本构建块，用于组织相关组件（控制器、服务等）。每个应用至少有一个根模块（AppModule）。

typescript
// src/app.module.ts
import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [],        // 导入其他模块
  controllers: [AppController],  // 该模块包含的控制器
  providers: [AppService],       // 该模块提供的服务（可被注入）
  exports: [],        // 导出供其他模块使用的服务
})
export class AppModule {}

4.2 控制器（Controller） ​

控制器负责处理客户端请求并返回响应，通过装饰器定义路由。

typescript
// src/app.controller.ts
import { Controller, Get } from '@nestjs/common';
import { AppService } from './app.service';

@Controller()  // 控制器装饰器，可选指定基础路径（如 @Controller('users')）
export class AppController {
  // 注入 AppService
  constructor(private readonly appService: AppService) {}

  @Get()  // 处理 GET 请求，路径为基础路径 + 此处定义（空表示 /）
  getHello(): string {
    return this.appService.getHello();  // 调用服务的方法
  }
}

五、基础 API 开发 ​

5.1 路由定义 ​

使用 HTTP 方法装饰器（@Get()、@Post()、@Put()、@Delete() 等）定义路由：

typescript
import { Controller, Get, Post } from '@nestjs/common';

@Controller('users')  // 基础路由为 /users
export class UsersController {
  @Get()  // 处理 GET /users
  findAll() {
    return '返回所有用户';
  }

  @Get(':id')  // 处理 GET /users/:id（动态参数）
  findOne() {
    return '返回单个用户';
  }

  @Post()  // 处理 POST /users
  create() {
    return '创建用户';
  }
}

5.2 处理请求数据 ​

5.2.1 获取路由参数 ​

使用 @Param() 装饰器：

typescript
import { Controller, Get, Param } from '@nestjs/common';

@Controller('users')
export class UsersController {
  @Get(':id')
  findOne(@Param('id') id: string) {  // 获取 id 参数
    return `用户 ID: ${id}`;
  }

  // 获取所有参数
  @Get(':id/posts/:postId')
  findUserPost(@Param() params: { id: string; postId: string }) {
    return `用户 ID: ${params.id}, 文章 ID: ${params.postId}`;
  }
}

5.2.2 获取查询参数 ​

使用 @Query() 装饰器：

typescript
import { Controller, Get, Query } from '@nestjs/common';

@Controller('users')
export class UsersController {
  @Get()
  findAll(@Query('page') page: string, @Query('limit') limit: string) {
    return `第 ${page} 页，每页 ${limit} 条`;
  }

  // 获取所有查询参数
  @Get('search')
  search(@Query() query: any) {
    return `查询参数: ${JSON.stringify(query)}`;
  }
}

5.2.3 获取请求体 ​

使用 @Body() 装饰器（需配合 @Post() 等方法）：

typescript
import { Controller, Post, Body } from '@nestjs/common';

@Controller('users')
export class UsersController {
  @Post()
  create(@Body() createUserDto: any) {  // DTO 用于数据验证（后续讲解）
    return `创建用户: ${JSON.stringify(createUserDto)}`;
  }

  // 获取部分请求体
  @Post('profile')
  updateProfile(@Body('name') name: string) {
    return `更新姓名为: ${name}`;
  }
}

5.4 依赖注入 ​

Nest 的依赖注入系统让组件间解耦，通过构造函数注入依赖：

typescript
// 服务定义
import { Injectable } from '@nestjs/common';

@Injectable()
export class UsersService {
  findAll() {
    return ['用户1', '用户2'];
  }
}

// 控制器中注入
import { Controller, Get } from '@nestjs/common';
import { UsersService } from './users.service';

@Controller('users')
export class UsersController {
  // 注入 UsersService
  constructor(private readonly usersService: UsersService) {}

  @Get()
  findAll() {
    return this.usersService.findAll();  // 调用服务方法
  }
}

七、错误处理 ​

Nest 提供了统一的异常处理机制，可通过 throw 抛出内置异常：

typescript
import { Controller, Get, NotFoundException } from '@nestjs/common';

@Controller('users')
export class UsersController {
  @Get(':id')
  findOne(@Param('id') id: string) {
    if (id === '0') {
      throw new NotFoundException('用户不存在');  // 抛出 404 异常
    }
    return `用户 ID: ${id}`;
  }
}
