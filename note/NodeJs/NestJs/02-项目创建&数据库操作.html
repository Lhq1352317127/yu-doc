Nest.js 手把手带你实战：项目创建&数据库操作 ​

一、初识 Nest.js ​

Nest.js 是一个用于构建高效、可扩展的 Node.js 服务器端应用程序的开发框架，具有以下特点：

  • 原生支持 TypeScript，同时兼容 JavaScript
  • 可基于 Express 或 Fastify 构建，暴露底层框架 API
  • 融合 OOP（面向对象编程）、FP（函数式编程）和 FRP（函数响应式编程）思想
  • 提供模块化架构，便于维护和扩展

二、项目创建 ​

1. 环境准备 ​

  • 要求 Node.js ≥ 10.13.0（v13 除外）
  • 推荐使用 nvm 管理 Node.js 版本

四、路由与控制器 ​

1. 控制器基础 ​

通过 @Controller() 装饰器定义控制器，可指定路由前缀：

typescript
// app.controller.ts
import { Controller, Get } from '@nestjs/common';
import { AppService } from './app.service';

@Controller("app")  // 路由前缀为 "app"
export class AppController {
  constructor(private readonly appService: AppService) {}

  @Get()  // 处理 GET 请求，路径为 /app
  getHello(): string {
    return this.appService.getHello();
  }
}

2. HTTP 方法装饰器 ​

支持 @Get、@Post、@Put、@Delete 等装饰器，用于处理不同 HTTP 方法：

typescript
@Controller("posts")
export class PostsController {
  // 固定路径：GET /posts/list
  @Get("list")
  getList() { return "文章列表"; }

  // 通配符路径：匹配 GET /posts/user_xxx（如 /posts/user_123）
  @Get("user_*")
  getUser() { return "用户文章"; }

  // 带参数路径：PUT /posts/:id
  @Put(":id")
  update() { return "更新文章"; }
}

3. 创建实体（Entity） ​

实体对应数据库表，使用 @Entity() 装饰器定义：

typescript
// src/posts/posts.entity.ts
import { Column, Entity, PrimaryGeneratedColumn } from "typeorm";

@Entity("posts") // 对应数据库表名
export class PostsEntity {
  @PrimaryGeneratedColumn() // 自增主键
  id: number;

  @Column({ length: 50 }) // 字符串类型，长度 50
  title: string;

  @Column("text") // 文本类型
  content: string;

  @Column({ type: 'timestamp', default: () => "CURRENT_TIMESTAMP" })
  create_time: Date;
}

4. 实现 CRUD 操作 ​

（1）创建服务（Service） ​

bash
nest g service posts  # 生成 PostsService
1
typescript
// src/posts/posts.service.ts
import { Injectable, HttpException } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { PostsEntity } from './posts.entity';

@Injectable()
export class PostsService {
  constructor(
    @InjectRepository(PostsEntity)
    private readonly postsRepository: Repository<PostsEntity>,
  ) {}

  // 创建文章
  async create(post: Partial<PostsEntity>) {
    const { title } = post;
    if (!title) {
      throw new HttpException('缺少文章标题', 400);
    }
    const existPost = await this.postsRepository.findOne({ where: { title } });
    if (existPost) {
      throw new HttpException('文章已存在', 400);
    }
    return this.postsRepository.save(post);
  }

  // 获取文章列表
  async findAll() {
    return this.postsRepository.find();
  }

  // 其他方法：findById、update、remove 等（略）
}

1. 错误处理（过滤器） ​

创建全局异常过滤器，统一错误响应格式：

bash
nest g filter core/filter/http-exception
1
typescript
// src/core/filter/http-exception.filter.ts
import { ArgumentsHost, Catch, ExceptionFilter, HttpException } from '@nestjs/common';

@Catch(HttpException)
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const status = exception.getStatus();

    response.status(status).json({
      code: -1,
      message: exception.message,
      data: {}
    });
  }
}

typescript
// src/core/interceptor/transform.interceptor.ts
import { CallHandler, ExecutionContext, Injectable, NestInterceptor } from '@nestjs/common';
import { map, Observable } from 'rxjs';

@Injectable()
export class TransformInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(
      map(data => ({
        code: 0,
        msg: '请求成功',
        data
      }))
    );
  }
}

八、数据库操作注意事项 ​

  1. 实体同步风险：synchronize: true 会自动根据实体创建数据库表，但生产环境建议关闭，避免意外删除数据。

  2. 实体字段修改：修改实体字段（如长度、类型）可能导致数据丢失，建议通过数据库迁移工具（如 TypeORM migrations）处理。

  3. 实体注册：确保实体已在 TypeOrmModule.forFeature() 中注册，否则会提示“无仓库”错误。
