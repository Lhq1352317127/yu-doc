使用Redis ​

一、认识Redis ​

1. Redis是什么 ​

Redis是一个开源的基于内存的数据结构存储系统，可作为数据库、缓存和消息中间件，具有以下特性：

  • 速度快：基于内存运行，单节点读速约110000次/秒，写速约81000次/秒，由C语言实现，性能高效。
  • 持久化：支持RDB和AOF两种方式，异步将数据保存到硬盘。
  • 多数据结构：支持字符串、哈希、列表、集合、有序集合等。
  • 跨语言支持：兼容多种编程语言。

2. Redis典型使用场景 ​

  • 缓存：减轻数据库压力，加速数据访问。
  • 排行系统：利用有序集合实现排行榜（如商品销量、用户积分）。
  • 计数器：适用于播放量、浏览数等高频计数场景。
  • 消息弹幕：直播间弹幕、在线用户列表等实时场景。

2. 配置Redis模块 ​

（1）创建Redis缓存模块 ​

typescript
// src/db/redis-cache.module.ts
import { ConfigModule, ConfigService } from '@nestjs/config';
import { RedisCacheService } from './redis-cache.service';
import { CacheModule, Module, Global } from '@nestjs/common';
import * as redisStore from 'cache-manager-redis-store';

@Module({
  imports: [
    CacheModule.registerAsync({
      isGlobal: true, // 全局模块，无需重复导入
      imports: [ConfigModule],
      inject: [ConfigService],
      useFactory: async (configService: ConfigService) => ({
        store: redisStore,
        host: configService.get('REDIS_HOST'), // 从环境变量获取
        port: configService.get('REDIS_PORT'),
        db: 0, // 目标数据库（默认0-15共16个库）
        auth_pass: configService.get('REDIS_PASSPORT'), // 密码（可选）
      }),
    }),
  ],
  providers: [RedisCacheService],
  exports: [RedisCacheService],
})
export class RedisCacheModule {}

（2）实现Redis缓存服务 ​

typescript
// src/db/redis-cache.service.ts
import { Injectable, Inject, CACHE_MANAGER } from '@nestjs/common';
import { Cache } from 'cache-manager';

@Injectable()
export class RedisCacheService {
  constructor(
    @Inject(CACHE_MANAGER)
    private cacheManager: Cache,
  ) {}

  // 存储数据到Redis（ttl单位：秒）
  cacheSet(key: string, value: string, ttl: number) {
    this.cacheManager.set(key, value, { ttl }, (err) => {
      if (err) throw err;
    });
  }

  // 从Redis获取数据
  async cacheGet(key: string): Promise<any> {
    return this.cacheManager.get(key);
  }
}

三、Redis优化Token管理 ​

1. Token过期处理 ​

登录时将JWT生成的token存入Redis，并设置有效期（如30分钟）。

typescript
// auth.service.ts
import { Injectable } from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';
import { RedisCacheService } from '../db/redis-cache.service';
import { User } from '../user/entities/user.entity';

@Injectable()
export class AuthService {
  constructor(
    private jwtService: JwtService,
    private redisCacheService: RedisCacheService,
  ) {}

  // 生成token并存储到Redis
  async login(user: Partial<User>) {
    const token = this.jwtService.sign({
      id: user.id,
      username: user.username,
      role: user.role,
    });

    // 存储token到Redis，有效期30分钟（1800秒）
    await this.redisCacheService.cacheSet(
      `${user.id}&${user.username}&${user.role}`, // 自定义key
      token,
      1800,
    );

    return { token };
  }
}

2. Token验证与过期检查 ​

验证时从Redis获取token，若不存在则判定为过期。

typescript
// jwt.strategy.ts
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { PassportStrategy } from '@nestjs/passport';
import { Strategy, ExtractJwt } from 'passport-jwt';
import { ConfigService } from '@nestjs/config';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { User } from '../user/entities/user.entity';
import { RedisCacheService } from '../db/redis-cache.service';

@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor(
    @InjectRepository(User)
    private userRepository: Repository<User>,
    private configService: ConfigService,
    private redisCacheService: RedisCacheService,
  ) {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      secretOrKey: configService.get('SECRET'),
      passReqToCallback: true, // 允许将请求传递给validate方法
    });
  }

  async validate(req, user: User) {
    // 从请求头提取token
    const token = ExtractJwt.fromAuthHeaderAsBearerToken()(req);
    // 从Redis获取缓存的token
    const cacheToken = await this.redisCacheService.cacheGet(
      `${user.id}&${user.username}&${user.role}`,
    );

    // 验证token是否存在（过期）
    if (!cacheToken) {
      throw new UnauthorizedException('token已过期');
    }

    // 验证token是否匹配（防止重复登录）
    if (token !== cacheToken) {
      throw new UnauthorizedException('token不正确');
    }

    // 验证用户是否存在
    const existUser = await this.userRepository.findOne({ where: { id: user.id } });
    if (!existUser) {
      throw new UnauthorizedException('token不正确');
    }

    return existUser;
  }
}

typescript
// jwt.strategy.ts（在validate方法中添加续期逻辑）
async validate(req, user: User) {
  // ... 省略前面的验证逻辑

  // 续期：重新设置token有效期为30分钟
  this.redisCacheService.cacheSet(
    `${user.id}&${user.username}&${user.role}`,
    token,
    1800,
  );

  return existUser;
}

四、总结 ​

通过Redis在Nest.js中实现了：

  • Token过期管理：利用Redis的过期机制自动失效token。
  • 自动续期：用户活跃时刷新token有效期，提升体验。
  • 唯一登录：新登录覆盖旧token，确保账号唯一在线。

Redis的高性能和丰富的数据结构使其成为缓存、计数、会话管理等场景的理想选择，结合Nest.js的模块化架构，可高效优化应用性能。
