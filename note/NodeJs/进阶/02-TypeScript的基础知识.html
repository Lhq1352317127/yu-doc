基础知识 ​

一、什么是 TypeScript？ ​

TypeScript（简称 TS）是微软开发的一门编程语言，它是 JavaScript 的超集（JavaScript + 类型系统）。这意味着所有 JavaScript 代码都可以在 TypeScript 中运行，同时 TypeScript 增加了静态类型检查功能。

为什么要用 TypeScript？ ​

  • 类型安全：在代码运行前就能通过类型检查发现错误（比如传参类型错误），减少生产环境 bug。
  • 更好的开发体验：IDE 支持自动补全、接口提示、重构建议等。
  • 代码可维护性：类型定义相当于「自文档」，让团队协作和后期维护更轻松。
  • 渐进式 adoption：可以逐步在 JavaScript 项目中引入 TypeScript，无需一次性重构。

三、核心语法：类型系统 ​

TypeScript 的核心是类型系统，通过为变量、函数、对象等添加类型注解（Type Annotation），实现静态类型检查。

3.1 基本类型 ​

TypeScript 支持 JavaScript 所有原始类型，并添加了类型注解语法（变量名: 类型）。

typescript
// 字符串
const name: string = "Alice";
const multiLine: string = `第一行
第二行`;

// 数字（包括整数、浮点数、NaN、Infinity）
const age: number = 25;
const pi: number = 3.14;
const notANumber: number = NaN;

// 布尔值
const isActive: boolean = true;
const hasError: boolean = false;

// 空值（表示没有返回值的函数）
function logMessage(): void {
  console.log("这是一个日志");
}

// Null 和 Undefined（默认是所有类型的子类型）
const empty: null = null;
const uninitialized: undefined = undefined;

// 任意类型（关闭类型检查，不推荐频繁使用）
let randomValue: any = "hello";
randomValue = 123;       // 不报错
randomValue = true;      // 不报错

3.4 接口（Interface） ​

接口用于定义对象的结构（属性和方法的类型），是 TypeScript 中最常用的类型定义方式之一。

typescript
// 定义用户接口
interface User {
  id: number;          // 必选属性
  name: string;        // 必选属性
  age?: number;        // 可选属性（? 表示可选）
  readonly role: string;  // 只读属性（初始化后不能修改）
}

// 使用接口约束对象
const user: User = {
  id: 1,
  name: "Alice",
  role: "admin"  // 只读属性
};

user.age = 25;    // 正确（可选属性可添加）
user.role = "user";  // 报错：只读属性不能修改

// 接口描述函数类型
interface AddFunction {
  (a: number, b: number): number;  // (参数: 类型) => 返回值类型
}

const add: AddFunction = (x, y) => x + y;
add(1, 2);  // 正确，返回 3
add("1", 2);  // 报错：参数类型不匹配

3.5 类型别名（Type Alias） ​

为类型创建一个别名，功能类似接口，但更灵活（可表示基本类型、联合类型等）。

typescript
// 为基本类型创建别名
type ID = string | number;
let userId: ID = "user123";
userId = 456;  // 正确

// 为对象类型创建别名
type Point = {
  x: number;
  y: number;
};

const point: Point = { x: 10, y: 20 };

// 接口 vs 类型别名：
// 1. 接口可被继承和实现，类型别名不行
// 2. 接口支持声明合并（多次定义会合并），类型别名不行
interface Animal {
  name: string;
}
interface Animal {  // 声明合并
  legs: number;
}
const dog: Animal = { name: "dog", legs: 4 };  // 正确

typescript
// 可选参数（必须放在必选参数后面）

5.2 访问修饰符 ​

控制类成员的访问权限：

  • public：默认，公开访问（类内、类外、子类均可访问）
  • private：仅类内可访问
  • protected：类内和子类可访问
typescript
class Student {
  public name: string;    // 公开
  private grade: number;  // 私有
  protected school: string;  // 受保护

  constructor(name: string, grade: number, school: string) {
    this.name = name;
    this.grade = grade;
    this.school = school;
  }

  private getGrade(): number {  // 私有方法
    return this.grade;
  }
}

class PrimaryStudent extends Student {
  constructor(name: string, grade: number, school: string) {
    super(name, grade, school);
  }

  getSchool(): string {
    return this.school;  // 正确：子类可访问 protected 成员
  }

  getGrade(): number {
    // return this.grade;  // 报错：子类不能访问 private 成员
    return 0;
  }
}

const student = new Student("Charlie", 5, "Sunny School");
console.log(student.name);  // 正确：public 成员可访问
// console.log(student.grade);  // 报错：private 成员不可访问

5.3 类实现接口 ​

类可以通过 implements 关键字实现接口，确保类符合接口定义的结构。

typescript
interface Movable {
  speed: number;
  move(): void;
}

// 类实现接口
class Car implements Movable {
  speed: number;  // 必须实现接口的属性

  constructor(speed: number) {
    this.speed = speed;
  }

  move(): void {  // 必须实现接口的方法
    console.log(`Car moving at ${this.speed} km/h`);
  }
}

const car = new Car(60);
car.move();  // 输出：Car moving at 60 km/h

七、类型断言与类型守卫 ​

当 TypeScript 无法推断类型时，可通过类型断言或类型守卫手动指定类型。

7.1 类型断言（Type Assertion） ​

告诉编译器「我知道这个值的类型」（类似类型转换，但不影响运行时）。

typescript
// 语法1：值 as 类型
const someValue: any = "this is a string";
const strLength: number = (someValue as string).length;

// 语法2：<类型>值（JSX 中不推荐，会与标签冲突）
const strLength2: number = (<string>someValue).length;

// 实际场景：获取 DOM 元素
const input = document.getElementById("username") as HTMLInputElement;
input.value = "Alice";  // 正确：知道 input 是输入框元素

7.2 类型守卫（Type Guard） ​

通过条件判断缩小类型范围，常用 typeof、instanceof、自定义函数。

typescript
// 1. 使用 typeof 检查基本类型
function printValue(value: string | number) {
  if (typeof value === "string") {
    console.log("String:", value.toUpperCase());  // 此处 value 被推断为 string
  } else {
    console.log("Number:", value.toFixed(2));     // 此处 value 被推断为 number
  }
}

// 2. 使用 instanceof 检查类实例
class Dog {
  bark() { console.log("Woof!"); }
}
class Cat {
  meow() { console.log("Meow!"); }
}

function makeSound(animal: Dog | Cat) {
  if (animal instanceof Dog) {
    animal.bark();  // 推断为 Dog
  } else {
    animal.meow();  // 推断为 Cat
  }
}

八、实战示例：用户管理 ​

综合运用上述知识，实现一个简单的用户管理功能：

typescript
// src/user.ts

// 1. 定义用户接口
interface User {
  id: string | number;  // 联合类型：ID 可以是 string 或 number
  name: string;
  age?: number;         // 可选属性
  readonly createdAt: Date;  // 只读属性
}

// 2. 泛型工具函数：创建用户
function createUser<T extends User>(userData: T): T {
  return {
    ...userData,
    createdAt: new Date()  // 补充创建时间
  };
}

// 3. 类：用户管理
class UserManager {
  private users: User[] = [];  // 私有成员：用户数组

  // 添加用户
  addUser(user: Omit<User, "createdAt">): User {  // Omit：排除 createdAt（TS 内置工具类型）
    const newUser = createUser({ ...user, createdAt: new Date() });
    this.users.push(newUser);
    return newUser;
  }

  // 根据 ID 查询用户
  getUserById(id: User["id"]): User | undefined {  // 索引类型查询：获取 User 的 id 类型
    return this.users.find(u => u.id === id);
  }
}

// 4. 使用示例
const manager = new UserManager();

// 添加用户
const user1 = manager.addUser({ id: "1", name: "Alice", age: 25 });
const user2 = manager.addUser({ id: 2, name: "Bob" });  // 可选属性 age 可省略

// 查询用户
const foundUser = manager.getUserById("1");
console.log(foundUser);  // 输出用户信息（包含 createdAt）

九、常见问题与解决 ​

  1. “类型 any 的使用”警告
    尽量避免使用 any，可用 unknown 替代（unknown 更安全，需要类型断言才能使用）。

  2. 接口和类型别名的选择

    • 定义对象结构优先用 interface（支持继承和合并）。
    • 定义联合类型、交叉类型等优先用 type。

  3. 编译错误“找不到模块”
    安装对应类型声明文件：npm install @types/模块名 --save-dev（如 @types/node）。

十、学习资源 ​

  • 官方文档：TypeScript 官方手册（最权威，示例丰富）。
  • 练习项目：将现有 JavaScript 小项目改写成 TypeScript，逐步熟悉语法。
  • 工具推荐：使用 VS Code 开发（对 TS 支持最佳），安装 ESLint 插件加强类型检查。

总结 ​

TypeScript 通过静态类型系统为 JavaScript 增加了一层保障，虽然初期需要多写一些类型注解，但长期来看能显著提升代码质量和开发效率。
