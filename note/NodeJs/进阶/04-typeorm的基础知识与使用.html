TypeORM基础知识与案例 ​

结合 CSDN 博客中的实用技巧，本文将从项目初始化到实体关系映射，全面讲解 TypeORM 的核心用法，通过具体示例帮助你快速上手。

一、TypeORM 简介与安装 ​

TypeORM 是一个跨平台的 ORM 框架，支持 TypeScript/JavaScript，兼容 MySQL、PostgreSQL 等主流数据库，通过实体类映射数据库表，简化数据操作。

1.1 全局安装 TypeORM CLI ​

bash
npm install typeorm -g
1

注意：mac 可能需要 sudo 权限，Windows 建议用管理员身份运行终端。

二、快速生成项目（官方脚手架） ​

使用 TypeORM 提供的 init 命令快速创建标准化项目：

bash
typeorm init --name MyProject --database mysql
1
  • --name：项目名称（如 MyProject）
  • --database：指定数据库类型（如 mysql、postgres 等）

生成的项目结构如下：

MyProject/
├── src/
│   ├── entity/          # 实体类目录（数据库表映射）
│   │   └── User.ts      # 示例用户实体
│   ├── migration/       # 数据库迁移文件
│   ├── subscriber/      # 事件订阅器
│   ├── index.ts         # 入口文件
│   └── ormconfig.json   # 数据库配置文件
├── package.json
└── tsconfig.json

三、核心配置文件（ormconfig.json） ​

自动生成的 ormconfig.json 是数据库连接的核心配置，详解如下：

json
{
  "type": "mysql",                // 数据库类型
  "host": "localhost",            // 数据库地址
  "port": 3306,                   // 端口（MySQL默认3306）
  "username": "root",             // 数据库用户名
  "password": "123456",           // 密码
  "database": "test",             // 数据库名
  "synchronize": true,            // 开发环境：自动同步实体到数据库（生产禁用）
  "logging": ["query", "error"],  // 打印SQL日志（可选值：query/error/schema）
  "entities": ["src/entity/**/*.ts"],  // 实体类路径
  "migrations": ["src/migration/**/*.ts"], // 迁移文件路径
  "subscribers": ["src/subscriber/**/*.ts"],
  "cli": {
    "entitiesDir": "src/entity",  // CLI生成实体的目录
    "migrationsDir": "src/migration",
    "subscribersDir": "src/subscriber"
  }
}

四、实体类定义（数据库表映射） ​

实体类是 TypeORM 的核心，通过装饰器映射数据库表结构。以自动生成的 User.ts 为例扩展讲解：

4.1 基础实体示例 ​

typescript
// src/entity/User.ts
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm";

@Entity({ name: "users" }) // 映射表名（默认类名小写，建议显式指定）
export class User {
  // 自增主键（int类型）
  @PrimaryGeneratedColumn()
  id: number;

  // 详细字段配置
  @Column({
    type: "varchar",      // 字段类型（MySQL中对应varchar）
    length: 50,           // 长度
    nullable: false,      // 不允许为NULL
    unique: true,         // 唯一约束
    name: "first_name",   // 数据库字段名（默认与属性名一致）
    comment: "用户名字"    // 字段注释
  })
  firstName: string;

  @Column()
  lastName: string;

  @Column({
    type: "int",
    default: 0,           // 默认值
    comment: "年龄"
  })
  age: number;
}

4.2 常用字段装饰器属性 ​

属性说明
type数据库字段类型（如 varchar、int、datetime）
name数据库实际字段名（默认与类属性名一致）
nullable是否允许为 NULL（默认 false）
unique是否添加唯一约束
length字段长度（如 varchar(50)）
default默认值
comment字段注释（建议必写，方便维护）
select查询时是否默认返回该字段（默认 true）

六、实体关系映射（重点） ​

TypeORM 支持一对一、一对多、多对多等关系，以下是实际场景示例：

6.1 一对一（User 与 UserExtend） ​

用户详情表（UserExtend）：

typescript
// src/entity/UserExtend.ts
import { Entity, PrimaryGeneratedColumn, Column, OneToOne, JoinColumn } from "typeorm";
import { User } from "./User";

@Entity({ name: "user_extend" })
export class UserExtend {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({
    type: "varchar",
    length: 11,
    nullable: true,
    name: "mobile",
    comment: "手机号"
  })
  mobile: string;

  @Column({ name: "address", comment: "地址" })
  address: string;

  // 一对一关联 User
  @OneToOne(() => User, user => user.userDetail)
  @JoinColumn() // 表示该表将存储关联的外键（user_id）
  user: User;
}

文章表（Article）：

// src/entity/Article.ts
import { Entity, PrimaryGeneratedColumn, Column, ManyToOne, CreateDateColumn } from "typeorm";
import { User } from "./User";

@Entity({ name: "articles" })
export class Article {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ name: "title", comment: "标题" })
  title: string;

  @Column({ name: "content", comment: "内容", type: "text" })

  createTime: Date;

  // 多对一：多篇文章属于一个用户

6.3 多对多（Article 与 Tag） ​

多对多关系需要中间表关联（如 article_tag），示例如下：

标签表（Tag）：

typescript
// src/entity/Tag.ts
import { Entity, PrimaryGeneratedColumn, Column, ManyToMany, JoinTable } from "typeorm";
import { Article } from "./Article";

@Entity({ name: "tags" })
export class Tag {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ name: "tag_name", comment: "标签名" })
  tagName: string;

  // 多对多：一个标签关联多篇文章
  @ManyToMany(() => Article, article => article.tags)
  @JoinTable({ name: "article_tag" }) // 中间表名
  articles: Article[];
}

七、数据库迁移（生产环境必备） ​

生产环境禁用 synchronize: true，需用迁移工具管理表结构：

7.1 创建迁移 ​

bash
npx typeorm-ts-node-commonjs migration:create src/migration/init-users
1

7.2 编写迁移脚本 ​

typescript
// src/migration/1694788888888-init-users.ts
import { MigrationInterface, QueryRunner } from "typeorm";

export class InitUsers1694788888888 implements MigrationInterface {
  // 执行迁移（创建表）
  async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`
      CREATE TABLE \`users\` (
        \`id\` int NOT NULL AUTO_INCREMENT,
        \`first_name\` varchar(50) NOT NULL,
        \`last_name\` varchar(255) NOT NULL,
        \`age\` int DEFAULT 0,
        PRIMARY KEY (\`id\`)
      ) ENGINE=InnoDB
    `);
  }

  // 回滚迁移（删除表）
  async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`DROP TABLE \`users\``);
  }
}

八、总结 ​

本文结合官方脚手架与实体关系示例，讲解了 TypeORM 的核心用法：

  1. 通过 typeorm init 快速生成标准化项目；
  2. 实体类通过装饰器映射数据库表，灵活配置字段属性；
  3. 基于 EntityManager 或 Repository 实现 CRUD 操作；
  4. 掌握一对一、一对多、多对多关系映射，应对复杂业务场景；
  5. 迁移工具是生产环境管理表结构的关键。

通过实际项目练习这些知识点，你可以轻松应对大多数数据库操作场景。更多细节可参考 TypeORM 官方文档。
