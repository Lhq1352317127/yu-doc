Vue3 快速上手指南 ​

1. Vue3 简介 ​

1.1 基本信息 ​

  • 发布时间：2020年9月18日，发布3.0版本，代号「One Piece」
  • 社区贡献：经历4800+次提交、40+个RFC、600+次PR、300+贡献者
  • 最新版本：截止2023年10月，最新公开版本为3.3.4
  • 官方发版地址：Release v3.0.0 One Piece · vuejs/core

1.2 核心优势 ​

  • 性能提升：
    • 打包大小减少41%
    • 初次渲染快55%，更新渲染快133%
    • 内存减少54%
  • 源码升级：
    • 用Proxy替代defineProperty实现响应式
    • 重写虚拟DOM实现和Tree-Shaking
  • 更好支持TypeScript
  • 新特性：
    • Composition API（组合API）
    • 新内置组件（Fragment、Teleport、Suspense）
    • 新生命周期钩子
    • 移除keyCode作为v-on修饰符等

3. Vue3核心语法 ​

3.1 Options API 与 Composition API ​

  • Options API（Vue2）：数据、方法、计算属性等分散在data、methods、computed中，维护不便
  • Composition API（Vue3）：以函数方式组织代码，相关功能聚合，便于维护和复用
  • Composition API优势：解决Options API中功能代码分散的问题，使逻辑更集中

3.2 setup函数（Composition API入口） ​

  • 特点：
    • 返回的对象可直接在模板中使用
    • 内部this为undefined
    • 执行时机在beforeCreate之前
  • 语法糖（简化写法）：
vue
<template>
  <div class="person">
    <h2>姓名：{{name}}</h2>
    <button @click="changeName">修改名字</button>
  </div>
</template>

<!-- setup语法糖 -->
<script setup lang="ts" name="Person">
// 数据（非响应式，需配合ref/reactive）
let name = '张三'

// 方法
function changeName() {
  name = '李四'
}
</script>

3.3 响应式数据 ​

ref与reactive ​

  • ref：用于定义基本类型或对象类型响应式数据，访问需加.value
  • reactive：用于定义对象类型响应式数据，嵌套属性自动响应式
  • 区别与使用原则：
    • ref需用.value访问，reactive无需
    • 基本类型必须用ref
    • 层级较深的对象推荐用reactive
    • reactive重新分配新对象会失去响应式（可使用Object.assign）

3.4 生命周期钩子 ​

3.5 props（组件通信-父传子） ​

接收方式：

  1. 仅接收：defineProps(['list'])
  2. 限制类型：defineProps<{list: Person[]}>()
  3. 带默认值与必要性：
typescript
import { withDefaults, defineProps } from 'vue'
import { type Person } from '@/types'

// 接收+限制类型+默认值
let props = withDefaults(defineProps<{ list?: Person[] }>(), {
  list: () => [{ id: '01', name: '默认名称', age: 18 }]
})

4. 路由（Vue Router 4） ​

4.1 基础配置 ​

javascript
// 路由配置文件（router/index.ts）
import { createRouter, createWebHistory } from 'vue-router'
import Home from '@/pages/Home.vue'
import About from '@/pages/About.vue'

const router = createRouter({
  history: createWebHistory(), // history模式（无#）
  routes: [
    { path: '/home', component: Home },
    { path: '/about', component: About }
  ]
})

export default router

4.3 关键特性 ​

  • 工作模式：

javascript

// 跳转

router.replace('/about') // 替换历史记录

5.2 核心概念与使用 ​

  • state：存储状态（类似data）
  • getter：计算属性（类似computed）
  • action：处理业务逻辑（类似methods，支持异步）
typescript
// 定义store（store/count.ts）
import { defineStore } from 'pinia'

export const useCountStore = defineStore('count', {
  state() {
    return { sum: 6 } // 状态
  },
  getters: {
    doubleSum: (state) => state.sum * 2 // 计算属性
  },
  actions: {
    increment() { // 方法
      this.sum++
    }
  }
})

7.1 Teleport（瞬移组件） ​

vue

  <div class="modal" v-show="isShow">
    <h2>弹窗内容</h2>
    <button @click="isShow = false">关闭</button>
  </div>

7.2 Suspense（异步加载组件） ​

等待异步组件时显示加载状态：

vue
<template>
  <Suspense>
    <template #default>
      <AsyncComponent /> <!-- 异步组件 -->
    </template>
    <template #fallback>
      <h3>加载中...</h3> <!-- 加载状态 -->
    </template>
  </Suspense>
</template>

<script setup lang="ts">
import { defineAsyncComponent } from 'vue'
// 异步引入组件
const AsyncComponent = defineAsyncComponent(() => import('./AsyncComponent.vue'))
</script>
