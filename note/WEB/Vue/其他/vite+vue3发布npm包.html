使用Vite+Vue3构建组件库并发布到npm（支持TS类型自动导出） ​

本文详细介绍如何通过Vite和Vue3构建组件库或JS工具插件，并发布到npm，同时实现TypeScript类型自动导出。

一、新建项目 ​

  1. 创建基础项目
    使用Vite初始化Vue3+TypeScript项目：

    bash
    pnpm create vite
# 输入项目名（如：my-component-lib）
# 选择框架：Vue
# 选择变体：TypeScript
cd my-component-lib  # 进入项目目录
pnpm install         # 安装依赖

declare module "*.vue" {
  import { DefineComponent } from "vue"
  const component: DefineComponent<{}, {}, any>
  export default component

  Vue: any

  4. 全局导出配置
    packages/index.ts（整合所有组件，支持全局注册）：

    typescript
    // 导入组件
import elButton from './el-button/index'
// 组件列表
const components = [elButton]
// 按需导出
export { elButton }
// 全局注册方法
const install = function(Vue: any) {
  if ((install as any).installed) return
  (install as any).installed = true
  components.forEach(component => {
    Vue.component(component.name, component)
  })
}
// 支持CDN引入时自动注册
if (typeof window !== 'undefined' && window.Vue) {
  install(window.Vue)
}
// 导出默认对象（含install方法）
export default { install }

五、配置发布信息（package.json） ​

package.json需包含发布必要字段，示例：

json
{
  "name": "my-component-lib",  // 包名（需唯一，可在npmjs.com查询）
  "private": false,            // 设为false才能发布
  "version": "0.0.1",          // 版本号（每次发布需递增）
  "type": "module",
  "main": "dist/my-component-lib.umd.js",  // UMD入口
  "module": "dist/my-component-lib.es.js",  // ES模块入口
  "types": "dist/index.d.ts",  // TS类型声明入口
  "files": [                   // 需发布的文件
    "dist/*",
    "packages/*",
    "types/*"
  ],
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc && vite build",  // 打包命令
    "preview": "vite preview"
  },
  "dependencies": { "vue": "^3.2.47" },
  "devDependencies": {
    "@vitejs/plugin-vue": "^4.1.0",
    "sass": "^1.62.1",
    "typescript": "^5.0.2",
    "vite": "^4.3.9",

六、配置Vite打包（vite.config.ts） ​

配置vite.config.ts，指定打包为组件库：

typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from "path"

const resolve = (dir: string) => path.join(__dirname, dir)

export default defineConfig({
  plugins: [vue()],  // 支持Vue
  resolve: {
    alias: {
      "@": resolve("examples"),
      "packages": resolve("packages")
    }
  },
  build: {
    rollupOptions: {
      external: ['vue'],  // 排除Vue依赖（避免打包时重复包含）
      output: {
        globals: { vue: 'Vue' }  // UMD模式下Vue的全局变量名
      }
    },
    lib: {
      entry: 'packages/index.ts',  // 打包入口
      name: 'my-component-lib',    // 库名称
      fileName: (format) => `my-component-lib.${format}.js`  // 输出文件名
    }
  }
})

七、TS类型自动导出 ​

使用vite-plugin-dts自动生成类型声明文件：

  1. 安装插件

    bash
    pnpm i vite-plugin-dts -D
    1

  2. 配置插件
    修改vite.config.ts：

    typescript
    import dts from 'vite-plugin-dts'  // 导入插件

export default defineConfig({
  plugins: [vue(), dts()],  // 添加dts插件
  // 其他配置不变
})

八、发布到npm ​

  1. 切换npm源
    确保使用官方源（非镜像源）：

    bash
    npm config get registry  # 检查是否为 https://registry.npmjs.com
npm config set registry https://registry.npmjs.com  # 切换官方源
    1
    2

  2. 登录npm

    bash
    npm login  # 输入用户名、密码、邮箱（需在npmjs.com注册）
    1

  3. 发布包

    bash
    npm publish  # 发布成功后可在npmjs.com查看
    1

    注意：测试包需在72小时内删除（npm unpublish 包名@版本号 --force），正式包发布后不可删除。

总结 ​

通过Vite+Vue3构建组件库的核心步骤：

  1. 规范目录结构（packages存组件，examples存示例）；
  2. 组件导出需实现install方法（支持全局/按需引入）；
  3. 配置package.json和vite.config.ts确保打包正确；
  4. 使用vite-plugin-dts自动生成TS类型；
  5. 切换官方源并发布到npm。

按此流程可快速构建可复用的Vue组件库，并支持TypeScript类型提示。
