vue3 + uniapp + ts 拓展全局类型处理 ​

背景 ​

在使用 uniapp + TypeScript 开发时，由于 Vue3 中 this 的限制，全局挂载工具函数不够便捷。通过扩展 uni 全局对象挂载属性或方法是可行方案，但需解决 TypeScript 的类型检查报错和 API 提示问题。

处理步骤 ​

1. 创建类型声明文件 ​

  • 在 src 目录下新建 types 文件夹，用于存放通用类型声明。
  • 在 types 文件夹中创建 global.d.ts 文件，用于扩展全局类型。

目录结构示例：

src/
├── types/
│   └── global.d.ts  # 全局类型声明文件
├── api/
├── components/
├── pages/
├── utils/
├── App.vue
├── main.ts
└── ...

2. 配置 tsconfig.json ​

在 tsconfig.json 中添加声明文件路径，确保 TypeScript 能识别自定义类型：

json
{
  "extends": "@vue/tsconfig/tsconfig.json",
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "Node",
    "strict": true,
    "baseUrl": "./",
    "paths": {
      "@/*": ["src/*"]  // 路径别名，便于导入
    },
    "types": ["vite/client", "@dcloudio/types", "@uni-helper/uni-app-types"]  // 包含uniapp类型
  },
  "include": ["src/**/*.ts", "src/**/*.d.ts", "src/**/*.vue"]  // 包含声明文件
}

3. 扩展 Uni 全局类型 ​

在 global.d.ts 中扩展 Uni 接口，添加自定义属性或方法的类型：

typescript
// 导入原始Uni类型
import type { Uni as _Uni } from '@dcloudio/types';

// 扩展全局Uni接口
declare global {
  interface Uni extends _Uni {
    // 自定义全局工具对象$u，包含loading和toast方法
    $u: {
      loading(show: boolean, title?: string): void;  // 加载提示
      toast(title: string): void;  // 消息提示
    };
  }
}

4. 实现工具函数并挂载到 uni ​

在 src/utils 中定义工具函数，并挂载到 uni 全局对象：

（1）实现工具函数 ​

typescript
// src/utils/index.ts

/**
 * 显示/隐藏加载提示
 * @param show 是否显示
 * @param title 提示文字（默认"加载中..."）
 */
function loading(show: boolean, title = '加载中...') {
  if (show) {
    uni.showLoading({ title });
  } else {
    uni.hideLoading();
  }
}

/**
 * 显示消息提示
 * @param title 提示内容
 */
function toast(title: string) {
  uni.showToast({
    title,
    icon: 'none'  // 不显示图标
  });
}

// 挂载到uni的$u属性
export default {
  install() {
    if (!uni || typeof uni !== 'object') return;
    uni.$u = {
      loading,
      toast
    };
  }
};

5. 初始化挂载 ​

在入口文件 main.ts 中引用并初始化工具函数，确保挂载生效：

typescript
// src/main.ts
import { createSSRApp } from 'vue';
import App from './App.vue';
import utils from '@/utils';  // 导入工具函数
import pinia from '@/pinia';  // 状态管理（可选）

export function createApp() {
  const app = createSSRApp(App);
  app.use(pinia);  // 初始化状态管理
  app.use(utils);  // 挂载工具函数到uni
  return { app };
}

效果 ​

完成后，可在项目中直接通过 uni.$u 调用工具函数，且 TypeScript 会提供类型提示，无报错：

typescript
// 显示加载提示
uni.$u.loading(true, '加载中...');

// 2秒后隐藏加载提示
setTimeout(() => {
  uni.$u.loading(false);
  uni.$u.toast('加载完成');  // 显示消息提示
}, 2000);

总结 ​

通过扩展 Uni 接口和挂载工具函数，解决了 uniapp + TypeScript 中全局工具的类型提示问题，实现了便捷的全局调用，同时保持类型安全。该方案可灵活扩展更多全局方法，适用于大型项目开发。
